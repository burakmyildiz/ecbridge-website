Transaction Flow

1

Initiate Transfer

User deposits tokens into the Ethereum bridge contract

2

Achieve Finality

Wait for Ethereum block finalization (~15 minutes)

3

Generate Proof

Relayer creates Merkle Patricia Trie proof of deposit

4

Verify on Citrea

SP1-Helios verifies proof against light client state

5

Mint Tokens

Citrea contract mints wrapped tokens to recipient

Security Architecture

Cryptographic Layer

  • Zero-knowledge SNARK proofs for state verification
  • Merkle Patricia Trie proofs for transaction inclusion
  • BLS signature verification for consensus

Smart Contract Layer

  • Reentrancy guards on all external functions
  • Nonce-based replay attack prevention
  • Comprehensive input validation

Infrastructure Layer

  • Redundant relayer services with failover
  • Comprehensive error handling and recovery
  • Real-time monitoring and alerting

Attack Resistance

Immune to 51% attacks through cryptographic verification

No Custody Risk

Funds locked in smart contracts, not held by validators

Full Transparency

All operations verifiable on-chain with open-source code
